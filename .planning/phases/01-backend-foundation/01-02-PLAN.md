Implement JWT authentication layer per RESEARCH.md Section 5.

**JwtTokenService.java:**
- Use jjwt library (io.jsonwebtoken:jjwt-api 0.12.x)
- generateToken(Account account): Create JWT with subject=email, claim("accountId", id), claim("tier", tier), expiration=24h (per AUTH-02 for session persistence)
- validateToken(String token): Parse and validate signature, expiration
- getAuthentication(String token): Return Spring Security Authentication with account details
- Use application.yml for jwt.secret and jwt.expiration configuration
- Store secret in base64-encoded format for HS256

**ApiKeyService.java:**
- validateApiKey(String apiKey): Query AccountRepository by apiKey, return account if valid
- generateApiKey(Account account): Generate UUID-based API key, save to account
- This satisfies AUTH-03 (API keys for service-to-service authentication)

**SecurityConfig.java (per RESEARCH.md Section 5):**
- @Configuration @EnableWebSecurity
- SecurityFilterChain bean:
  - Disable CSRF (stateless API)
  - Stateless session management
  - Permit /api/auth/** endpoints
  - All other requests authenticated
  - Add JwtAuthenticationFilter before UsernamePasswordAuthenticationFilter
- PasswordEncoder bean using BCryptPasswordEncoder

**JwtAuthenticationFilter.java (per RESEARCH.md Section 5):**
- Extend OncePerRequestFilter
- Extract Bearer token from Authorization header
- Validate token via JwtTokenService
- Set Authentication in SecurityContextHolder
- Handle invalid/expired tokens gracefully (return 401)

**LoginRequest.java:**
- Fields: email (String, @NotBlank), password (String, @NotBlank)

**AuthResponse.java:**
- Fields: token (String), accountId (UUID), email (String), tier (AccountTier), expiresIn (Long)

Per AUTH-01: Login returns JWT token. Per AUTH-02: JWT persists across refresh (24h expiry).

**AccountRepository.java:**
- Extend JpaRepository<Account, UUID>
- findByEmail(String email): Optional<Account> (for login)
- findByApiKey(String apiKey): Optional<Account> (for API key validation)
- existsByEmail(String email): boolean

**AccountService.java:**
- @Service with AccountRepository dependency
- login(String email, String password): Validate credentials with BCrypt, generate JWT via JwtTokenService, return AuthResponse
- createAccount(AccountDto dto): Hash password, save account, generate API key
- getAccount(UUID id): Return account by ID
- getAccountByEmail(String email): Return account by email
- updateCreditLimit(UUID id, BigDecimal newLimit): Update credit limit
- updateBalance(UUID id, BigDecimal amount): Update current balance
- Use MapStruct or manual mapping for AccountDto conversion

**AuthController.java:**
- POST /api/auth/login: Accept LoginRequest, return AuthResponse with JWT
- POST /api/auth/register: Accept AccountDto, create account, return AuthResponse
- This satisfies AUTH-01 (login returns JWT)

**AccountController.java:**
- GET /api/accounts/{id}: Get account by ID (authenticated)
- GET /api/accounts: List all accounts (authenticated, paginated)
- POST /api/accounts: Create new account (authenticated)
- PUT /api/accounts/{id}/credit-limit: Update credit limit
- PUT /api/accounts/{id}/balance: Update balance
- This satisfies ACCT-01 (credit limits, balance) and ACCT-02 (tier determines pricing)

**AccountDto.java:**
- Fields: email, password, companyName, tier, creditLimit
- Use validation annotations (@NotBlank, @Email)

**GlobalExceptionHandler.java:**

- Handle MethodArgumentNotValidException (400)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → AuthController | Login credentials cross from untrusted to trusted |
| JWT → Service | Token claims used for authorization decisions |
| API Key → Service | Service-to-service authentication via API key header |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | JWT token | mitigate | Validate signature, expiration, and claims in JwtAuthenticationFilter |
| T-02-02 | Tampering | Password storage | mitigate | BCrypt hashing, never store plaintext |
| T-02-03 | Information Disclosure | JWT secret | mitigate | Store in environment variable, not in code |
| T-02-04 | Elevation of Privilege | Account creation | accept | Open registration for MVP, restrict in Phase 2 |
| T-02-SC | Tampering | Maven dependencies | mitigate | Only use jjwt official library, not custom crypto |
</threat_model>
