Create JPA entities per the RESEARCH.md schema design. All entities use UUID primary keys (GeneratedValue UUID), Lombok annotations (@Data, @Builder, @NoArgsConstructor, @AllArgsConstructor), and @Entity with @Table.

**Account Domain (account-service):**

AccountTier.java (enum): STANDARD, SILVER, GOLD, PLATINUM

Account.java:
- Fields: id (UUID), email (String, unique, not null), passwordHash (String, not null), companyName (String), tier (AccountTier, not null), creditLimit (BigDecimal), currentBalance (BigDecimal), apiKey (String, unique), createdAt (LocalDateTime), updatedAt (LocalDateTime)
- Use @Enumerated(EnumType.STRING) for tier
- Use @PrePersist and @PreUpdate for timestamps
- This entity satisfies ACCT-01 (credit limits, balance) and ACCT-02 (tier determines pricing)

**Catalog Domain (catalog-service):**

Category.java:
- Fields: id (UUID), name (String, not null), parentId (UUID, self-referencing), sortOrder (Integer, default 0)
- Use @ManyToOne for parentId relationship (optional)

Product.java:
- Fields: id (UUID), sku (String, unique, not null), name (String, not null), description (String/Text), basePrice (BigDecimal, not null), inventoryLevel (Integer, default 0), categoryId (UUID, foreign key), isActive (Boolean, default true), createdAt (LocalDateTime), updatedAt (LocalDateTime)
- Use @ManyToOne for categoryId relationship
- This entity satisfies CATALOG-01 (SKU, name, description, price, inventory)

TierPricing.java:
- Fields: id (UUID), productId (UUID, foreign key), tier (AccountTier, not null), price (BigDecimal, not null)
- Use @Enumerated(EnumType.STRING) for tier
- Use @UniqueConstraint on (productId, tier)
- This entity satisfies CATALOG-03 (tier-based pricing)

**Order Domain (order-service):**

OrderStatus.java (enum): PENDING, CONFIRMED, SHIPPED, DELIVERED

Order.java:
- Fields: id (UUID), accountId (UUID, not null), status (OrderStatus, not null), totalAmount (BigDecimal), creditUsed (BigDecimal), createdAt (LocalDateTime), updatedAt (LocalDateTime)
- Use @Enumerated(EnumType.STRING) for status
- Use @OneToMany for orderItems with CascadeType.ALL, orphanRemoval
- This entity satisfies ORDER-01 (order creation) and ORDER-02 (status tracking)

OrderItem.java:
- Fields: id (UUID), order (Order, @ManyToOne), productSku (String, not null), productName (String), quantity (Integer, not null), unitPrice (BigDecimal, not null), totalPrice (BigDecimal, not null)
- Denormalize product data per RESEARCH.md recommendation
- This entity satisfies ORDER-01 (line items with SKU + quantity)

All entities must follow the schema from RESEARCH.md Section 2 exactly. Do not deviate from the specified field names, types, or relationships.

| Client → Gateway | External HTTP requests cross into internal service mesh |
| Gateway → Services | Internal routing with JWT validation |
| Service → Database | Each service owns its database, no cross-access |
| Service ↔ Service | Event-driven messaging via RabbitMQ |
