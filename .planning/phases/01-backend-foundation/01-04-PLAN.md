Implement the order business logic, REST endpoints, event publishing, and seed data.

**OrderEventPublisher.java:**
- @Service with StreamBridge dependency
- publishOrderCreated(OrderCreatedEvent event): Send to "orderCreated-out-0" channel
- publishOrderStatusChanged(OrderStatusChangedEvent event): Send to "orderStatusChanged-out-0" channel
- Per D-03, D-04: Event-driven messaging via Spring Cloud Stream

**OrderService.java:**
- @Service with OrderRepository, OrderItemRepository, OrderEventPublisher dependencies
- createOrder(CreateOrderRequest request): 
  1. Validate account exists (call account-service via REST or Feign client)
  2. Validate products exist and have sufficient inventory (call catalog-service)
  3. Calculate total amount from line items
  4. Create Order entity with PENDING status
  5. Create OrderItem entities for each line item
  6. Save order and items
  7. Publish OrderCreatedEvent
  8. Return OrderResponse
- getOrder(UUID id): Return order with items
- getOrdersByAccount(UUID accountId, int page, int size): Paginated order history
- updateOrderStatus(UUID id, OrderStatus newStatus):
  1. Find order, validate status transition (PENDING -> CONFIRMED -> SHIPPED -> DELIVERED)
  2. Update status
  3. Publish OrderStatusChangedEvent
  4. Return updated OrderResponse
- getOrderStatus(UUID id): Return just the status (for chatbot queries per CHAT-01)

**OrderController.java:**
- POST /api/orders: Create new order (authenticated)
- GET /api/orders/{id}: Get order by ID (authenticated)
- GET /api/orders: List orders with optional accountId filter, paginated
- PUT /api/orders/{id}/status: Update order status (authenticated)
- GET /api/orders/{id}/status: Get order status only (for chatbot)
- This satisfies ORDER-01 (create with line items), ORDER-02 (status tracking), ORDER-03 (view history)

**GlobalExceptionHandler.java:**
- @RestControllerAdvice
- Handle ResourceNotFoundException (404)
- Handle InsufficientInventoryException (400)
- Handle InvalidOrderStatusException (400)
- Return structured error responses

**data.sql (seed data):**

  4. Order for admin@acme.com: 1 item, CONFIRMED status
  5. Order for buyer@globex.com: 4 items, PENDING status
- Insert corresponding order items with product SKUs from catalog seed data

Per D-05: Events published for order creation and status changes.
