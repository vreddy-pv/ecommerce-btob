  <action>
Implement the catalog business logic and REST endpoints.

**CatalogService.java:**
- @Service with ProductRepository, CategoryRepository, TierPricingRepository dependencies
- getProducts(String search, UUID categoryId, int page, int size): Paginated product list with optional search/filter
- getProductBySku(String sku): Single product with tier pricing
- getProductWithTierPrice(String sku, AccountTier tier): Product with tier-specific price (fall back to basePrice if no tier pricing)
- getCategories(): List of root categories with children
- getCategoryById(UUID id): Single category with products
- createProduct(ProductDto dto): Save new product
- updateInventory(String sku, int quantity): Update inventory level (called by order service via events)
- Use Pageable for pagination per RESEARCH.md recommendation

**CatalogController.java:**
- GET /api/catalog/products: List products with optional search, category, pagination
  - Query params: search (String), categoryId (UUID), page (int, default 0), size (int, default 20)
- GET /api/catalog/products/{sku}: Get product by SKU with tier pricing
- GET /api/catalog/products/{sku}/price?tier={tier}: Get tier-specific price
- GET /api/catalog/categories: List all categories
- GET /api/catalog/categories/{id}: Get category by ID with products
- POST /api/catalog/products: Create new product (authenticated)
- PUT /api/catalog/products/{sku}/inventory: Update inventory level

**GlobalExceptionHandler.java:**
- @RestControllerAdvice
- Handle ResourceNotFoundException (404)
- Handle MethodArgumentNotValidException (400)
- Return structured error responses

Per CATALOG-01: Products display with all required fields. Per CATALOG-02: Search and filter work. Per CATALOG-03: Tier pricing returns correct discounted prices.
