# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Angular Material 17 |
| Preset | Material Design 3 (MD3) |
| Component library | Angular Material (@angular/material) |
| Icon library | Material Icons (@angular/material) |
| Font | Roboto (Google Fonts, bundled with Angular Material) |

**Rationale:** Angular Material is the official Angular component library. MD3 provides a mature, accessible design system suitable for B2B enterprise dashboards. Roboto is the default Material font — clean and functional for data-heavy interfaces.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Card padding, section padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, page header to content |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: none

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (Regular) | 1.5 |
| Label | 12px | 500 (Medium) | 1.4 |
| Heading | 20px | 500 (Medium) | 1.3 |
| Display | 28px | 400 (Regular) | 1.2 |

**Font Family:** Roboto, system-ui, sans-serif

**Rationale:** 14px body text is standard for B2B dashboards with dense data tables. Display is reserved for page-level titles only.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (White) | Page background, card surfaces |
| Secondary (30%) | `#F5F5F5` (Grey 100) | Sidebar, table alternating rows, navigation background |
| Accent (10%) | `#1976D2` (Blue 700) | Primary action buttons, active nav items, status links |
| Destructive | `#D32F2F` (Red 700) | Delete actions, error states, destructive confirmations |

**Accent reserved for:**
- Primary CTA buttons ("Place Order", "Apply Filters")
- Active navigation indicator
- Clickable status badges (links to order detail)
- Form field focus borders
- **Never** used for: backgrounds, text on white, borders

**Status Colors (semantic):**
- Pending: `#F9A825` (Amber 700) — order pending status badge
- Shipped: `#1976D2` (Blue 700) — order shipped status badge
- Delivered: `#388E3C` (Green 700) — order delivered status badge
- Error: `#D32F2F` (Red 700) — API errors, validation failures

## Component Inventory

### Catalog View (CATALOG-01, CATALOG-02, CATALOG-03)

| Component | Material Component | Purpose |
|-----------|-------------------|---------|
| Search bar | `mat-form-field` + `matInput` | Search by name, SKU, description |
| Filter chips | `mat-chip-listbox` | Category filter (multi-select) |
| Product card | Custom (mat-card wrapper) | Display part: image, SKU, name, price, inventory |
| Price display | Custom | Show tier-specific pricing based on account level |
| Inventory badge | `mat-badge` | Low stock warning (<10 units) |
| Sort dropdown | `mat-select` | Sort by name, price, SKU |

**Product Card Layout:**
- Product image (placeholder if none)
- Part name (Label, 14px, 500 weight)
- SKU (Label, 12px, grey)
- Price (Body, 14px, 500 weight, accent color for tier pricing)
- Inventory level (Label, 12px, green/red based on stock)

### Orders View (ORDER-01, ORDER-02, ORDER-03)

| Component | Material Component | Purpose |
|-----------|-------------------|---------|
| Orders table | `mat-table` | Order history with columns |
| Status chip | Custom (mat-chip) | PENDING/SHIPPED/DELIVERED badges |
| Order actions | `mat-icon-button` | View detail, track order |
| Date range filter | `mat-date-range-input` | Filter orders by date |
| Create order FAB | `mat-fab` | Floating action button to start new order |

**Orders Table Columns:**
1. Order ID (clickable link)
2. Date Placed
3. Items Count
4. Total Amount
5. Status (colored chip)
6. Actions (icon buttons)

### Order Creation Flow (ORDER-01)

| Component | Material Component | Purpose |
|-----------|-------------------|---------|
| Cart sidebar | Custom panel | Running order total |
| Quantity input | `mat-form-field` + `matInput` | Quantity per line item |
| Add to order | `mat-raised-button` | Add part to order |
| Submit order | `mat-raised-button` (accent) | Finalize and submit |

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Click product card | Navigate to product detail (optional, not in Phase 2 MVP) |
| Click "Add to Order" | Add item to cart sidebar, show snackbar confirmation |
| Click "Place Order" | Open confirmation dialog with order summary |
| Submit order | POST to API, show success snackbar, navigate to Orders view |
| Filter changes | Debounced API call (300ms), update product grid without page reload |
| Table row click | Expand row or navigate to order detail (Phase 2 MVP: highlight row) |
| Status chip click | (Future: link to tracking page) |

---

## Accessibility Requirements

- All interactive elements must have visible focus indicators
- Color is never the sole indicator of state (status chips include text labels)
- Table headers use `scope="col"` for screen readers
- Form fields use `mat-label` for associated labels
- Keyboard navigation: Tab through cards, Enter to select, Escape to close dialogs
- ARIA labels on icon buttons (e.g., `aria-label="View order details"`)
