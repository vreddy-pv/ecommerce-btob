<objective>
Build the catalog browsing feature: CatalogService (HTTP calls to /api/catalog/products and /api/catalog/categories), ProductCardComponent (single part display with tier-specific pricing), and CatalogComponent (search bar + category filter chips + sort dropdown + product grid with 300ms debounced search). Purpose: This is the primary landing view of the B2B dashboard. Without it, users cannot browse parts to add to orders.
Output: A functional catalog page at /catalog showing a searchable, filterable, tier-priced product grid.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Spoofing | CatalogService.getProducts params | mitigate | Use Angular HttpParams (encodes values); backend validates search/categoryId params (Jakarta Validation in Phase 1) |
| T-02-08 | Tampering | ProductCardComponent product data display | mitigate | Angular template binding ({{ }}) auto-sanitizes; never bind product.name/description to innerHtml — use text interpolation only |
