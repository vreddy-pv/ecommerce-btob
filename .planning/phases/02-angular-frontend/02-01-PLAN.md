The gateway is WebFlux-based (Spring Cloud Gateway), so use a reactive `RouterFunction<ServerResponse>` bean — NOT the servlet `WebMvcConfigurer` (that is blocking/Tomcat only and will not apply under WebFlux/Netty).

Create `gateway-service/src/main/java/com/btob/gateway/config/StaticResourceConfig.java` as a @Configuration class. Inject a resource location via @Value("${btob.static-resource-location:classpath:/static/}") so the location is configurable (classpath for embedded builds, file: path for Docker volume mounts). The class exposes TWO beans:
