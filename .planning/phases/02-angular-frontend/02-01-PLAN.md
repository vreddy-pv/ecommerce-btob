<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Gateway API | Untrusted HTTP requests cross here; JWT in Authorization header is the credential |
| localStorage → AuthService | Token read from localStorage on page load; XSS could exfiltrate |
| Angular dev server (:4200) → Gateway (:8080) | Cross-origin in dev without proxy/CORS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | AuthService login | mitigate | Login form uses Angular reactive forms with Validators.email + minLength; backend verifies BCrypt password (Phase 1) |
| T-02-02 | Tampering | JWT in localStorage | mitigate | Angular built-in sanitization prevents XSS in templates; never use innerHTML without sanitization; token is sent only via Authorization header (immune to CSRF) |
| T-02-03 | Elevation | authGuard | mitigate | CanActivateFn checks isAuthenticated() signal on every navigation to /catalog, /orders, /orders/:id; redirects to /login if no token |
| T-02-04 | Information Disclosure | errorInterceptor | mitigate | On 401, logout + redirect; never surface raw backend stack traces — display parsed ApiError.message only (per UI-SPEC copywriting) |
| T-02-05 | Tampering | gateway CORS | mitigate | CorsWebFilter allows ONLY http://localhost:4200 (not wildcard) in dev; production uses same-origin static serving per D-04 (no CORS needed) |
| T-02-06 | Spoofing | open redirect after login | mitigate | Router only navigates to known routes (/catalog, /orders, /login); no user-controlled redirect URLs |
| T-02-SC | Tampering | npm package installs (Angular 22 released 4 days ago) | accept | All @angular/* packages flagged SUS "too-new" but are official Angular org first-party packages (github.com/angular/angular) with millions of weekly downloads; no postinstall scripts; legitimacy audit approved — false positive on too-new flag for major framework release |
</threat_model>
